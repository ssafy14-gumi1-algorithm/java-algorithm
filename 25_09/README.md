## 커리큘럼 왜 이렇게 만듦?
8월에는 알고리즘의 개념과 그것을 템플릿처럼 적용하면 바로 풀 수 있는 문제를 유형별로 매주 풀었다.\
9월에는 문제가 어떤 유형인지 알아내는 능력을 기르기 위해 랜덤 유형의 문제를 매주 3문제 낼 예정\
아직 못 배운 알고리즘이 많으나, 시뮬레이션, 완탐, 재귀, dfs, bfs 유형의 어려운 문제를 풀 수 있게 되는 게 우선순위가 높다고 판단했다.
