# 요구사항 분석
게임판의 상태가 한 줄로 주어지면, 그 상태가 틱택토 게임에서 발생할 수 있는 최종 상태인지 판별하는 것

## 입력
입력은 여러 개의 테스트 케이스로 이루어져 있다.\
각 줄은 9개의 문자를 포함하며, 'X', 'O', '.' 중 하나이다.\
'.'은 빈칸을 의미하며, 9개의 문자는 게임판에서 제일 윗 줄 왼쪽부터의 순서이다.\
입력의 마지막에는 문자열 "end"가 주어진다.

## 출력(목표)
각 테스트 케이스마다 한 줄에 정답을 출력한다.\
가능할 경우 "valid", 불가능할 경우 "invalid"를 출력한다.

# 왜 코드를 그렇게 짰는지
1. 일단 O, X, 빈칸의 개수를 세어줌
   - O와 X의 개수가 같거나 X의 개수가 하나 더 많아야함
   - 빈칸의 개수를 센 이유? 둘 다 이기지 않았을 때, 게임판이 꽉찼는지 확인하기 위해서
2. O와 X가 각각 이겼는지 확인
3. O와 X가 둘 다 이겼으면 유효하지 않은 게임판
4. O가 이겼다면 O와 X의 개수가 같아야 함
5. X가 이겼다면 X가 O보다 하나 많아야함
6. 둘 다 진 경우
   - 게임판이 꽉 차 있다면 유효한 게임판

# 핵심 로직
유효한 게임판인지 확인하는 조건문
```declarative
if (cntX == cntO || cntX == cntO + 1) {
    // 둘 다 이기는 경우
    if (winX && winO) {
        valid = false;
    }
    // X만 이김 -> X가 한 수 더 많아야 함
    else if (winX) {
        if (cntX == cntO + 1) valid = true;
    }
    // O만 이김 -> X와 O 수가 같아야 함
    else if (winO) {
        if (cntX == cntO) valid = true;
    }
    // 아무도 안 이김 -> 보드가 꽉 차 있고, X가 한 수 더 많아야 게임 끝
    else {
        if (cntDot == 0 && cntX == cntO + 1) valid = true;
    }
}
```
